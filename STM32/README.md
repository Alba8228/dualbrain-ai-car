# 智能小车 - STM32实时控制模块

## 项目简介

本模块是智能小车项目的实时控制核心，基于STM32F103微控制器，负责舵机云台驱动、传感器采集、实时运动控制等功能。通过USART1与OpenWRT通信，接收AI控制指令并执行相应动作。

## 硬件平台

- **主控芯片**: STM32F103R8
- **开发环境**: Keil MDK-ARM
- **外设库**: STM32F10x_StdPeriph_Driver

## 核心功能

### 1. 蓝牙遥控模式
- 通过HC-05蓝牙模块接收手机App指令
- 支持前进、后退、左转、右转、停止
- 三挡速度可调(低/中/高)
- 控制LED、蜂鸣器、舵机

### 2. 超声波避障模式
- HC-SR04超声波测距
- 三方向扫描策略(前/左/右)
- 自动判断障碍物并规避
- 舵机云台旋转探测

### 3. 红外循迹模式
- 5路红外循迹传感器
- 自动识别黑线轨迹
- 差速控制实现转向

### 4. AI跟随模式
- 接收OpenWRT转发的人体检测框信息
- 根据检测框位置和面积控制小车跟随
- 支持多目标跟踪

### 5. 自动模式
- 综合多种传感器信息
- 自主决策运动方向

### 主循环逻辑
```c
while(1)
{
    Key_Shift();              // 按键模式切换
    switch(key_mode)
    {
        case 1: 蓝牙遥控模式(); break;
        case 2: 超声波避障模式(); break;
        case 3: 红外循迹模式(); break;
        case 4: AI跟随模式(); break;
        case 5: 自动模式(); break;
    }
}
```

## 通信机制

### USART1 DMA + 空闲中断
这是STM32与OpenWRT通信的关键，解决了串口丢包问题。

**特点**:
- DMA自动接收，无需CPU干预
- 空闲中断触发表示一帧数据接收完成
- 支持不定长数据
- 高效实时，减少CPU开销

**数据接收流程**:
1. DMA自动从USART1接收数据存入缓冲区
2. 串口空闲时触发中断
3. 计算已接收数据长度
4. 主循环解析指令帧
5. 校验通过后执行动作，否则请求重传

### 指令帧格式
- 帧头: `AA 55`
- 指令字节: 小车方向(00-04) + 舵机方向(05) + 速度(0A-0C)
- 校验和: 所有字节之和取低8位

## 驱动模块API

### 电机驱动 - motor.c
| 函数 | 功能 |
|------|------|
| `CAR_Motor_Config()` | GPIO+TIM PWM初始化 |
| `motor_go(speed_l, speed_r)` | 前进，左右轮调速 |
| `motor_back(speed)` | 后退 |
| `motor_left(speed)` | 原地左转 |
| `motor_right(speed)` | 原地右转 |
| `motor_stop()` | 四轮停止 |

### 超声波测距 - sr04.c
| 函数 | 功能 |
|------|------|
| `Sr04_Init()` | 超声波初始化 |
| `GetLength_Sr04()` | 获取距离值(cm) |

### 舵机控制 - sg90.c
| 函数 | 功能 |
|------|------|
| `Steer_Init()` | 舵机PWM初始化 |
| `Steer_Control()` | 控制舵机角度 |

### LCD显示 - lcd.c
| 函数 | 功能 |
|------|------|
| `LCD_Init()` | LCD初始化 |
| `LCD_Fill()` | 填充矩形 |
| `LCD_ShowString()` | 显示字符串 |
| `LCD_ShowChinese()` | 显示中文 |
| `LCD_ShowPicture()` | 显示图片 |

## 编译与烧录

1. 使用Keil MDK-ARM打开 `project/CarProject.uvprojx`
2. 选择目标芯片 STM32F103R8
3. 点击编译按钮或按F7
4. 连接ST-Link调试器
5. 点击下载按钮烧录程序
